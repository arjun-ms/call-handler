# Changelog - 2026-05-15T20:33:34

## 🎯 Focus: Initial Implementation & Core Pipeline

### ✅ What Worked
- **FastAPI Core:** The service structure is solid, with middleware handling request timing and unique IDs effectively.
- **Audio Standardization:** Using `ffmpeg` for normalization solved the issue of supporting multiple codecs and sample rates. The 16kHz mono PCM standard is now the reliable baseline for all downstream tasks.
- **VAD-based Early Stopping:** Successfully implemented speech extraction limits in `app/audio/vad.py`. This significantly improves latency and memory safety for long-form audio by only processing the first few seconds of actual speech.
- **Dual Model Strategy:** The implementation of both `audonnx` (for performance) and `transformers` (as a fallback) ensures the service remains functional even if optimized runtimes are unavailable.
- **CREMA-D Integration:** Successfully established a test harness that validates the pipeline against a real dataset with ground-truth labels.

### ❌ What Didn't (and Why)
- **Direct PyTorch Inference Latency:** Initial tests showed that standard PyTorch inference was too slow for the 500ms target on CPU.
  - **Resolution:** Pivoted to ONNX (`audonnx`), which provides a significant speedup for the `wav2vec2` architecture.
- **webrtcvad Compilation:** Encountered issues installing `webrtcvad` on some environments due to missing C++ build tools.
  - **Resolution:** Implemented a robust `_energy_vad` fallback to ensure the service stays alive even without the specialized library.
- **Model Download Timeouts:** Large model weights sometimes caused timeouts during startup.
  - **Resolution:** Added explicit preloading in the FastAPI `lifespan` event and documented the cache volume in Docker Compose.

### 🛠️ Key Decisions
- **Age Brackets over Scalar Age:** Decided to predict broad ranges (e.g., 18-30, 31-45) because voice-based age estimation is inherently noisy. This improves the "perceived accuracy" and stability of the system.
- **Confidence-Aware Responses:** Implemented an `unknown` fallback for low-confidence predictions to prevent misleading users in noisy environments.
- **Temporary File Strategy:** Chose to use `tempfile` with immediate `cleanup_temp` instead of in-memory bytes to handle larger uploads without spiking RAM usage.
